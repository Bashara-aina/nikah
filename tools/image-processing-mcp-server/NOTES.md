# image-processing-mcp-server (nikah fork notes)

Project-local checkout of
[`rafael-castelo/image-processing-mcp-server`](https://github.com/rafael-castelo/image-processing-mcp-server)
wired into Cursor via `~/.cursor/mcp.json`.

This is **not** a Node dependency of `nikah-web/` — it runs as a separate
stdio process spawned by Cursor.

## What it gives us

6 tools exposed in chat:

| Tool | Purpose |
| --- | --- |
| `resize-image` | Resize by width / height, optional aspect-ratio lock |
| `compress-image` | Reduce file size, lossless or quality-controlled |
| `convert-image-format` | jpeg, png, webp, avif, tiff |
| `get-image-metadata` | Size, format, dimensions, color space |
| `crop-image` | Rectangular crop |
| `batch-image-processing` | Run many ops in one call |

All ops target `sharp`, so they're fast and lossless where the format allows.

## Why a project-local clone (not `npx -y …`)

- **Reproducible** — pinned to the commit we tested, not whatever npm serves today.
- **Offline-friendly** — works without network on subsequent boots.
- **No npx cache churn** — first run is instant after `npm install`.

## Why invoke `build/index.js` directly (not the `bin`)

The package ships a `bin: image-processing-mcp-server` entry, but calling
`build/index.js` directly:

- avoids needing the bin on `PATH`,
- makes the exact entry point visible in `mcp.json`,
- is what the upstream README documents under "LOCAL".

## Setup on a fresh machine

```bash
cd tools/image-processing-mcp-server
npm ci            # or: npm install
npm run build
```

The build script does `tsc && chmod 755 build/index.js`. The `build/` output
is gitignored; the server is unusable without it.

`~/.cursor/mcp.json` should **not** contain this server — it lives in the
project-level config so the team shares it via git:

`.cursor/mcp.json` (in the repo root):

```json
{
  "mcpServers": {
    "image-processing": {
      "command": "node",
      "args": ["tools/image-processing-mcp-server/build/index.js"],
      "cwd": "${workspaceFolder}"
    }
  }
}
```

`${workspaceFolder}` is expanded by Cursor to the project root at spawn time,
so the config works regardless of where the repo is checked out.

## Upgrading

```bash
cd tools/image-processing-mcp-server
git fetch origin
git pull --ff-only           # or: git checkout <tag>   for a pinned version
rm -rf node_modules build
npm install
npm run build
```

Then restart Cursor (or toggle the server off/on) so it re-spawns the stdio
process.

## Smoke test

From inside the checkout, with `node_modules` populated:

```bash
node -e '
import("@modelcontextprotocol/sdk/client/index.js").then(async ({ Client }) => {
  const { StdioClientTransport } = await import("@modelcontextprotocol/sdk/client/stdio.js");
  const t = new StdioClientTransport({
    command: "node",
    args: ["./build/index.js"],
  });
  const c = new Client({ name: "smoke", version: "0" }, { capabilities: {} });
  await c.connect(t);
  const { tools } = await c.listTools();
  console.log(tools.map(x => x.name).join(","));
  await c.close();
});
'
```

Expected: `resize-image,compress-image,get-image-metadata,convert-image-format,crop-image,batch-image-processing`.

## Full validation

For more than a smoke test — exercises every tool on real project assets,
tests all 6 format conversions, batch processing, and error handling:

```bash
npm run validate
```

Runs `validate.mjs`, which boots the server exactly as the project-level
`mcp.json` does (relative path, `cwd: ${workspaceFolder}`), runs **51
assertions** across 9 test groups, and exits non-zero on any failure.
Last verified result: `SUMMARY: 51/51 passed, 0 failed`.

## Cursor config check

Confirms `.cursor/mcp.json` is present, valid, and would launch correctly:

```bash
npm run verify:cursor
```

Runs `verify-cursor-config.mjs`, which reads the project config, expands
`${workspaceFolder}`, spawns the server from a deliberately foreign cwd to
prove the relative path is what makes the config portable, and verifies
the global `~/.cursor/mcp.json` does **not** also define the same server
(single source of truth).
